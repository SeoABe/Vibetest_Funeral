# MVP 사용자 시나리오

## 전제

이 시나리오는 실제 서비스 출시가 아니라 바이브코딩 실증을 위한 MVP 테스트용이다. 모든 인물과 사건은 허구이며, 이미지도 실존 인물을 묘사하지 않는 추모용 플레이스홀더다.

## Persona A: 대표 상주 김도윤

- 나이: 38세
- 관계: 고인의 장남
- 목표: 모바일로 빠르게 부고를 만들고 가족 단톡방과 지인에게 공유한다.
- 주요 니즈: 입력이 어렵지 않아야 하고, 장례식장 위치와 조의금 계좌가 정확히 보여야 한다.

### 시나리오 A-1: 부고 생성

1. 관리자가 `/admin/login`에 접속한다.
2. 테스트 관리자 계정으로 로그인한다.
3. `부고 생성`을 누른다.
4. 고인 정보에 `김하루`를 입력하고 가상 사진 `deceased-kim-haru.png`를 선택한다.
5. 상주 정보에 배우자, 자녀, 사위/며느리, 손주 정보를 입력한다.
6. 장례 일정에 입관, 발인, 장지 정보를 입력한다.
7. 장례식장 검색 결과에서 `한빛추모병원 장례식장`을 선택한다.
8. 조의금 계좌 2개를 등록한다.
9. 방명록은 `관리자 승인 후 공개`로 설정한다.
10. 비공개 추모글 비밀번호를 설정한다.
11. 미리보기에서 고인명, 발인일, 장례식장, 계좌 복사 버튼이 보이는지 확인한다.
12. 공개 처리 후 카카오톡 공유 링크를 복사한다.

### 성공 기준

- 공개 URL이 생성된다.
- 부고 상단에서 발인일, 장례식장, 대표 상주가 즉시 보인다.
- 지도, 계좌, 방명록으로 이동하는 액션이 보인다.
- 계좌번호는 버튼을 누른 후 표시된다.
- 공유 이미지가 `og-kim-haru.svg`로 연결된다.

## Persona B: 조문객 박서연

- 나이: 45세
- 관계: 고인의 전 직장 동료
- 목표: 카카오톡으로 받은 부고를 열고, 장례식장 위치와 발인 시간을 확인한 뒤 방명록을 남긴다.

### 시나리오 B-1: 부고 열람과 방명록 작성

1. 카카오톡 메시지에서 부고 링크를 연다.
2. 첫 화면에서 고인명, 발인일, 장례식장명을 확인한다.
3. `장례식장 보기` 버튼을 누른다.
4. 주소 복사 버튼을 눌러 주소 복사 성공 상태를 확인한다.
5. `조의금 계좌` 섹션에서 계좌 표시 버튼을 누른다.
6. 계좌 복사 버튼을 누른다.
7. 방명록 섹션으로 이동한다.
8. 이름 `박서연`, 메시지 `삼가 고인의 명복을 빕니다. 따뜻했던 마음 오래 기억하겠습니다.`를 작성한다.
9. 작성 후 `관리자 승인 후 공개됩니다` 안내를 확인한다.

### 성공 기준

- 모바일 화면에서 버튼 터치 영역이 충분하다.
- 방명록 작성 직후 공개 목록에는 보이지 않는다.
- 관리자 승인 대기 목록에 새 메시지가 추가된다.

## Persona C: 가족 이민재

- 나이: 33세
- 관계: 고인의 손자
- 목표: 가족에게만 공유된 비공개 추모글을 비밀번호로 열람한다.

### 시나리오 C-1: 비공개 추모글 열람

1. 공개 부고 페이지 하단의 `가족에게 남긴 글` 섹션으로 이동한다.
2. 잘못된 비밀번호를 입력한다.
3. 실패 안내가 과도하게 구체적이지 않은지 확인한다.
4. 테스트 비밀번호 `haru-family-0427`을 입력한다.
5. 비공개 추모글 내용을 확인한다.
6. 새로고침 후 다시 잠금 상태가 유지되는지 확인한다.

### 성공 기준

- 실패 시 비밀번호 존재 여부나 내부 상태를 노출하지 않는다.
- 성공 시 테스트용 추모글이 표시된다.
- rate limit 테스트를 위해 반복 실패 입력이 가능하다.

## Persona D: 운영 관리자 정유진

- 나이: 29세
- 역할: MVP 운영자
- 목표: 부고 목록을 관리하고 방명록을 승인/삭제한다.

### 시나리오 D-1: 방명록 승인

1. `/admin/dashboard`에 접속한다.
2. 승인 대기 방명록 수를 확인한다.
3. `김하루` 부고의 방명록 관리 화면으로 이동한다.
4. 정상 메시지 2개를 승인한다.
5. 테스트용 부적절 메시지 1개를 삭제한다.
6. 공개 페이지에서 승인된 메시지만 보이는지 확인한다.

### 성공 기준

- 승인/삭제 후 목록 상태가 즉시 반영된다.
- 삭제된 메시지는 공개 페이지에 보이지 않는다.
- 관리자 작업 실패 시 재시도 안내가 나온다.

## 테스트 흐름 추천

1. `김하루` 데이터로 핵심 플로우를 먼저 구현한다.
2. `이정민` 데이터로 공개/비공개 상태, 이미지 없는 케이스, 일정이 일부 비어 있는 케이스를 검증한다.
3. 방명록 승인 대기, 승인 완료, 삭제 후보 상태를 모두 화면에 노출해본다.
4. 카카오톡 공유는 실제 SDK 연동 전에는 `asset-manifest.json`의 OG 이미지 경로로 미리보기 UI만 검증한다.
